자, 너는 지금부터 우주 최고의 구글 Apps Sciprt 웹 앱 개발자다! 

나는 시설에 있는 HAVC공조장비를 관리하는 관리자 겸 점검자야. 항상 수기로 장비의 내용을 기입하고 관리하다 보니 불편한 점이 많아서 이번에 새로운 프로젝트로  "장비 관리 앱"을 만들려고 해. 

장비에 해당 큐알을 부착하여서 점검자는 점검을 진행.

 공조장비 관련자 혹은 현장담당자가 접속하면 통계나 점검 내역을 쉽게 볼 수 있게! 

앱 주요 기능과 사용자 흐름은 다음과 같아. 

== [앱 주요 기능] 시작 == 

1. 점검자가 현장명을 선택,해당 현장에 해당되는 건물명 선택,해당 건물의 층과 위치,장비종류를 선택하고, 해당장비의 점검내용을 입력하면 구글시트 폼에 자동으로 입력하는 웹 앱을 만들고 싶어 (보통 모바일을 사용하여 확인)

2. 구글 스프레드시트 = DB, Apps Script = 서버, HTML = 프론트엔드 (로그인 없이 내부 사용자용 웹앱) 구조를 생각하고있고 여기저기 물어보니 내가 가지고있는 점검항목(기존 엑셀 혹은 스프레드시트의 헤더 수)이 많아서 일반점검과 세부점검을 구분해서 만들어야 좋을 것 같다고 하더라 더 좋은방법이 있다면 말해줘 

== [앱 주요 기능] 끝.

== == [사용자 흐름] 시작 == 

1. 점검자가 점검하는 장비를 선택하여 점검내용을 기입(점검내용은 천천히 수정해서 만들어보자)

2. HVAC업무 관련자가 차트 / 이상 건물 강조 /  기간별 필터 전부 포함된 완성형 통계 화면

3. [점검완료] 버튼을 클릭하면 해당 장비의 점검내용을 기록

 == [사용자 흐름] 끝. == 

데이터는 구글 시트에서 장비점검시트로 관리할 거야.
